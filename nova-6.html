Core Features

⚙️
Config-Driven Experiments

Dynamic config validation with cfshow and sanity_check. Keep configs stateless, fill values at runtime. Inspect subtrees on demand.

🔧
Multi-Task Orchestration

Flexible launch tooling with mp_run.py. Single-task SFT or full multi-process RL pipelines. Override any param from CLI.

🧩
Modular Stacks

Extensible data, optimizer, and model stacks for rapid research iteration. Plug in custom kernels — Flash Attention, grouped GEMM, Triton — only when needed.

🤖
AI-Native Workflow

Ships with AGENTS.md to guide AI agents. Modular architecture supports easy verification and iterative development with LLMs.

⚡
Optimization Kernels

Optional operator-level replacements for maximum GPU performance. Flash-Attn, NV grouped GEMM, and torch_compile — set all at once with set_optimization().

🌐
Distributed Rendezvous

Per-experiment Redis server spun up automatically via shared filesystem. Multi-node scripts generated by build_scripts.py with per-replica shell scripts.

Dev Workflow

01
DEFINE CONFIG
Declare config structure at class level. Use Ref("..path") for cross-node linkage. Configs stay stateless.
02
INSPECT & VALIDATE
Run cfshow to overview the experiment and sanity-check before launching. Inspect subtrees with -k flag.
03
LAUNCH EXPERIMENT
Single-task via torchrun or multi-task via mp_run.py. Override any param inline from CLI without editing files.
04
SCALE TO MULTI-NODE
Generate per-replica shell scripts with build_scripts.py. Each script exports task envs and invokes the assembled command.
nova_experiment.py 
# Define a typed config
from configurize import Config

class TrainConfig(Config):
    model_name: str = "qwen3-8b"
    lr: float = 1e-4
    batch_size: int = 32

    def sanity_check(self):
        super().sanity_check()
        assert self.lr > 0
        assert self.batch_size >= 1

# Inspect before running
cfshow playground/rlvr/qwen3_1p5b.py

# Launch multi-task RL
tools/mp_run.py playground/rlvr/qwen3_1p5b.py \
  profiler_cfg.timing_log_level=1

# Enable kernel-level speedups
from nova.utils.optimizable import set_optimization

set_optimization(
    default="torch_compile",
    AttentionCore="flash-attn",
    grouped_gemm="nv_grouped_gemm",
)
03 ——

Training Modes

Supervised
SFT Training

Supervised fine-tuning with modular data pipelines. Reference configs available for Qwen3 8B and large-scale architectures.

  • Multi-format data preparation
  • Dynamic config validation
  • torchrun single-node launch
  • Muon optimizer support
Reinforcement
RLVR Pipeline

Reinforcement Learning from Verifiable Rewards. Multi-task orchestration with distributed rendezvous via Redis across all nodes.

  • Multi-node multi-process launch
  • Shared filesystem rendezvous
  • Per-replica shell script gen
  • CLI param overrides
Pre-training
Pre-Train Configs

Reference configurations for large-scale pre-training. Supports Qwen3 and other open model families out of the box.

  • Qwen3 8B reference config
  • Custom architecture configs
  • Modular model stack
  • Custom optimizer plugins
Evaluation
Eval Workflows

Integrated evaluation pipelines for benchmarking across training runs. Config-driven and reproducible by default.

  • Configurable eval schedule
  • Reproducible benchmarks
  • Triton kernel integration
  • Flash-Attn acceleration

Quick
Install

Only PyTorch required as a base dependency. Optimization kernels are optional and installed separately for maximum GPU performance.

uv sync
1
Clone the repository
git clone https://github.com/nova-ai/nova
2
Install with uv
cd nova && uv sync
3
Optional: Flash Attention
uv pip install flash-attn
4
Run your first experiment
cfshow playground/rlvr/qwen3_1p5b_rlvr_math.py
